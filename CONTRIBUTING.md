## 貢献の種類

### 1. 新しい情報の提供

報道記事、公的発表、遺族の発信など、新しい情報を見つけた場合:

- **方法:** [Issueを作成](../../issues/new?template=new-information.md)
- **テンプレート:** 「新しい情報の提供」を使用
- **必須情報:** 日付、情報源、内容の要約

### 2. 誤報・デマの報告

誤った情報やデマを発見した場合:

- **方法:** [Issueを作成](../../issues/new?template=misinformation-report.md)
- **テンプレート:** 「誤報・デマの報告」を使用
- **必須情報:** 問題のある内容、正しい情報、根拠

### 3. 日次更新の編集

毎日の更新ファイルを直接編集する場合:

- **方法:** Pull Requestを作成
- **対象:** `updates/YYYY-MM/YYYY-MM-DD.md`
- **要件:** 情報源の明記、客観的な記述

### 4. レポート本体の修正

完全版レポートの誤字脱字や事実誤認の修正:

- **方法:** Pull Requestを作成
- **対象:** `henoko-media-bias-report.md`
- **要件:** 修正理由の明確な説明

## 情報提供の方法

### ステップ1: 情報の確認

提供する前に以下を確認してください:

- [ ] 情報源が明確である
- [ ] 日付が特定できる
- [ ] 内容が具体的である
- [ ] デマや誤報でないことを確認した

### ステップ2: Issueの作成

1. [Issues](../../issues) ページを開く
2. 「New issue」をクリック
3. 適切なテンプレートを選択
4. すべての項目を記入
5. 「Submit new issue」をクリック

### ステップ3: フォローアップ

- Issueへのコメントを確認
- 追加情報の提供が求められた場合は対応
- 情報が反映されたらIssueがクローズされます

## 更新ファイルの編集

### GitHub Desktopを使う場合

#### 1. リポジトリのフォーク

1. GitHubのリポジトリページで「Fork」をクリック
2. 自分のアカウントにフォークを作成

#### 2. GitHub Desktopでクローン

1. GitHub Desktopを起動
2. 「File」→「Clone Repository」
3. フォークしたリポジトリを選択

#### 3. ファイルを編集

1. `updates/YYYY-MM/YYYY-MM-DD.md` を開く
2. 情報を追加・更新
3. 情報源のURLを必ず記載

#### 4. コミット

1. GitHub Desktopで変更を確認
2. コミットメッセージを記入（例: 「2026-04-23: 産経新聞の記事を追加」）
3. 「Commit to main」をクリック

#### 5. プッシュとPull Request

1. 「Push origin」をクリック
2. GitHubのページで「Pull Request」を作成
3. 変更内容を説明して送信

### Webブラウザで直接編集する場合

1. GitHubのファイルページを開く
2. 鉛筆アイコン（Edit）をクリック
3. ファイルを編集
4. ページ下部で変更内容を説明
5. 「Propose changes」→「Create pull request」

## 品質基準

### 情報源の信頼性

#### ✅ 高信頼性（優先的に採用）
- 公的機関の公式発表
- 遺族の直接的な証言（note等）
- 主要メディアの署名記事
- 専門家のコメント（実名）

#### ⚠️ 要検証（確認後に採用）
- 匿名の情報提供
- SNSの情報
- 週刊誌の記事
- まとめサイト

#### ❌ 除外（原則として採用しない）
- 根拠不明の情報
- デマ・誹謗中傷
- 遺族のプライバシーを侵害する情報
- 政治的プロパガンダ

### 記述の基準

#### 客観性
```markdown
✅ 良い例: 「産経新聞は4月21日、学校側が救命胴衣の確認を行っていなかったと報じた」
❌ 悪い例: 「学校は無責任にも救命胴衣の確認をサボった」
```

#### 情報源の明記
```markdown
✅ 良い例: 「遺族のnoteによれば、知華さんは『サンゴ礁を見たい』と話していたという」
❌ 悪い例: 「知華さんはサンゴ礁を見たかっただけだった」
```

#### 推測と事実の区別
```markdown
✅ 良い例: 「報道量の少なさから、メディアが意図的に報道を控えている可能性がある」
❌ 悪い例: 「メディアは明らかに報道を隠蔽している」
```

## 重要な注意事項

### 🚨 絶対に守るべきルール

#### 1. 遺族のプライバシー保護

- ❌ 遺族の住所、電話番号、勤務先などの個人情報を記載しない
- ❌ note以外の私的な情報（家族写真、私信など）を掲載しない
- ✅ 遺族が公開している情報のみを扱う
- ✅ 遺族の意向を最優先する

#### 2. 被害者の尊厳の尊重

- ❌ 武石知華さんの写真を無断で使用しない
- ❌ センセーショナルな表現を避ける
- ✅ 知華さんの人物像を正確に伝える
- ✅ 敬意を持った表現を心がける

#### 3. 政治的中立性

- ❌ 特定の政治的立場を推進しない
- ❌ 「基地反対派が悪い」「政府が悪い」といった単純化を避ける
- ✅ 複数の視点を提示する
- ✅ 事実を客観的に記録する

#### 4. 情報の正確性

- ❌ 未確認の情報を事実として記載しない
- ❌ 憶測を断定的に書かない
- ✅ 情報源を必ず明記する
- ✅ 不確かな情報には「要確認」「報道による」などを付記

### 📝 推奨される記述スタイル

#### 日付の記載
```markdown
✅ 2026年4月22日
✅ 2026-04-22
❌ 4/22（年の記載がない）
```

#### 情報源の記載
```markdown
✅ 産経新聞（2026年4月21日）によれば、
✅ [産経新聞の記事](URL)
❌ ネットニュースによれば、
```

#### 引用の方法
```markdown
✅ 遺族のnoteには「美しいサンゴ礁を友達と見たい」という知華さんの思いが綴られている。
❌ 知華さんは美しいサンゴ礁を友達と見たいと思っていた。（情報源不明）
```

1. Issueまたはコメントを報告

## 質問・サポート

### よくある質問

**Q: どのくらいの頻度で更新すべきですか？**
A: 新しい情報があればその都度。毎日の定期更新は必須ではありません。

**Q: 小さな情報でも提供していいですか？**
A: はい。些細に思える情報でも重要な場合があります。

**Q: 匿名で貢献できますか？**
A: GitHubアカウントは必要ですが、実名でなくても構いません。

**Q: 間違いを見つけたらどうすればいいですか？**
A: Issueで報告するか、直接Pull Requestで修正してください。
